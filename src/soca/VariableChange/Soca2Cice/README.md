# SOCA2CICE: Sea Ice Analysis Postprocessing for CICE Model

## Overview

The SOCA2CICE variable change post-processes total sea ice concentration analysis generated by SOCA to a format suitable for the CICE sea ice model. 

## Purpose

Sea ice in CICE is represented using multiple thickness categories (`ncat`), while SOCA analysis provides aggregated 2D fields. SOCA2CICE bridges this gap by:

- Distributing aggregated sea ice variables across CICE's categories
- Ensuring physical consistency of the sea ice fields
- Preventing model instabilities from unrealistic ice configurations

## Key Functionalities

### 1. Ice Bounds Checking (`check_ice_bounds`)
- Ensures sea ice variables stay within physical limits
- Corrects ice concentration to be between `min_aice` (by default 0) and 1
- Forces negative ice/snow thickness to 0

### 2. Ice Distribution Shuffling (`shuffle_ice`)
- Operates in the marginal ice zone (where concentration < `seaice_edge`)
- When analysis shows ice in areas without background ice:
  - Identifies nearby ice-covered background points with similar concentration
  - Copies the ice distribution profile to maintain realistic category distribution
- When analysis removes ice:
  - Sets all ice-related variables to zero
  - Optionally adjusts SST to more realistic values near ice edge

### 3. Prior Distribution Rescaling (`prior_dist_rescale`)
- Operates in the ice pack (where concentration > `seaice_edge`) 
- Rescales the background ice distribution to match the analyzed aggregated values
- Preserves the shape of the distribution across categories
- Separately adjusts ice thickness and snow thickness to match the analysis when they exceed minimum thresholds

### 4. Ice State Cleanup (`cleanup_ice`)
- Calls icepack's `cleanup_itd` to rebin thickness categories if necessary
- Removes ice with volume below a threshold (`min_vice`) to prevent model instability
- Removes ice over land points
- Recomputes aggregate fields for consistency

## Configuration Options

SOCA2CICE offers separate configurations for Arctic and Antarctic regions:

### Regional Parameters
- `seaice edge`: Concentration threshold defining the ice edge (default: 0.15)
- `shuffle`: Enable/disable shuffling in marginal ice zone (default: true)
- `rescale prior`: Options for rescaling in the ice pack
  - `rescale` : Enable/disable rescaling in the ice pack (default: false)
  - `min hice`: Minimum ice thickness to trigger adjusting ice thickness to the analysis ice thickness during rescaling (default: 0.5m)
  - `min hsno`: Minimum snow thickness to trigger adjusting snow thickness to the analysis snow thickness during rescaling (default: 0.1m)
- `update SST`: Adjust sea surface temperature when adding/removing ice (default: true)
- `max positive SST update`: Maximum allowed SST increase when removing ice (default: 1.0K)

### General Parameters
- `min_aice`: Minimum ice concentration (default: 0.0)
- `min_vice`: Minimum ice volume (default: 0.00001)
- `icepack time step`: Time step for thickness rebinning (default: 300s)
- `shuffle stencil depth`: Size of search area for similar ice points (default: model's halo size from the namelist)
- `icepack tfrz_option`: Option to compute freezing temperature in icepack ("mushy", "linear", ) (default: "mushy")

## Scientific Rationale

The transformation performed by SOCA2CICE is essential for preventing model instabilities and preserving physical consistency. Direct insertion of aggregated ice fields into CICE's multi-category framework can lead to:

1. Unrealistic category distributions
2. Thermodynamic inconsistencies
3. Discontinuities at the ice edge
4. Model crashes from invalid ice states

By maintaining the category structure from nearby points or the background state, SOCA2CICE preserves the realistic representation of ice thickness distribution that is important for accurate thermodynamic and dynamic calculations in CICE.
